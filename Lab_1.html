Preguntas y Respuestas HTML:

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
  • Internet es una red conformada por multiples redes más pequeñas, como WANS, MANS o LANS , interconectadas a un mismo eje; por otra parte, World Wide Web se define por Ramón Olivella en su libro Diseño y Programación de Aplicaciones Web como una infraestructura de la información para la física de altas energías, quiere decir, la estructura con la que se manejan los multiples datos en la Internet.
2. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
  • GET - Solicita una representación de un recurso específico, solo recuperan datos.
  • HEAD - Pide una respuesta idéntica a GET, pero sin el cuerpo de respuesta.
  • POST - Envía una entidad a un recurso en específico, generalmente causa un cambio de estado en el servidor.
  • PUT - Reemplaza todas las representaciones actuales de un recurso de destino.
  • PATCH - Aplica modificaciones parciales en un recurso.
  • DELETE - Borra un recurso en específico.
3. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
  • Se utiliza el método POST para poder actualizar el servidor sin exponer la información del usuario.
4. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
  • Se utiliza el método GET.
5. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
  • No hay error, el protocolo HTTP regresa un 200 cuando todo está OK
6. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
  • Puede ser, 404 es un error de Recurso no encontrado, y se ocasiona debido a que el usuario trata de acceder a un recurso temporalmente desabilitado, o inexistente.
7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
  • Si, el error 500 es Internal Server Error, normalmente avisa al administrador lo sucedido para prevenir que ocurra nuevamente

9. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
  • HTML5 contiene HTML(contenido y estructura), CSS(presentación) y JavaScript(comportamiento), mientras que en HTML4 se realiza todo con HTML.
10. ¿Qué componentes de estructura y estilo tiene una tabla?
  • Parte de los componentes de estructura que se presentan en las tablas de HTML5 son padding, titulos alineados, celdas de multiples columnas, celdas de multiples filas; mientras algunos de estilo pueden ser bordes, definición, espaciado de los bordes, color de fondo, color de fila, entre otros.

Preguntas y Respuestas CSS:

  1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    2. Tomar en cuenta si es necesario incluirlo o no, dado que por un malo uso de !important se puede asignar un estilo a un elemento en el que no se desea, y ocasionar errores muy sutiles.

  2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    3. Depediendo de la imagen selecionada, al momento de cambiar el tamaño de una ventana, se puede perder calidad o inclusive el formato de la misma puede no desplegarse dependiendo del navegador. Por eso es importante seleccionar una iagen que cumpla con el objetivo deseado.

  3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    4. % Es la medida más confiable a utilizar cuando queremos que la página se pueda adaptar al tamaño de la ventana. en cambio pt y px son fijas, y a no ser que se esté 100% seguro del tamaño, es mejor usar medidas adaptables.

  4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    5. Debido a que optimiza los tiempos de carga de la página

Glosario de Terminos
Concepto Definición
URL Uniform Resource Locator. Es como se le conoce a una dirección de una página en la WWW.
MIME Multipurpose Internet Mail Extensions. Un estándar para dar formato a diferentes archivos multimedia para que puedan ser transportados y vistos en los navegadores web o las aplicaciones de e-mail.
Métodos y Códigos de errores Método se refiere a cómo interactuarán los datos del navegador con el servidor, mientras que los Códigos de error son los resultantes de las interacciones.
HTTP Hyperlink Text Transfer Protocol. Protocolo utilizado para navegar en la WWW.
XML eXtended Mark-up Language. Lenguaje que permite asignar significado a datos para facilitar un dialogo entre aplicaciones. Se basa en separar la estructura de un documento en su contenido.
XHTML eXtensible HyperText Mark-up Language. es HTML definido como una aplicación XML. Es prácticamente igual a HTML pero más estricto, pues todos los documentos deben estar marcados correctamente, es decir, bien formados.
HTML5 HyperText Mark-up Language. Es el lenguaje utilizado hoy en día por la mayoría para estructurar y presentar el contenido de una página en la Web. Es el sucesor de HTML, con la principal diferencia de que hay más posibilidades de estructura y formato que con su antecesor, utilizando menos recursos.
JS Java Script. Es también un lenguaje de programación que se ejecuta en el equipo del cliente, no en el servidor. Utilizado para el comportamiento de ciertos elementos en una página web.
php Hypertext PreProcessor. Es un lenguaje de análisis de servidor, el más utilizado en desarrollo web.
jsp Java Server Pages. Es una manera de programar páginas web basandose en HTML/XML pero con algunas etiquetas de sintáxis de Java.
Ajax Asynchronous -javascriot And XML. Es una técnica para desarrollar sitios web, escrito en JavaScript e intercambiando la información con el servidor en XML de manera asíncrona. AJAX permite que una página web que ya ha sido cargada solicite información nuevamente.
